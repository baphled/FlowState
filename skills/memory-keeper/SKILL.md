# Memory-Keeper Skill

Systematically capture problem-solution pairs, patterns discovered, and common mistakes into a knowledge graph. This creates searchable institutional memory that prevents repeating debugging work and accelerates future problem-solving.

Core principle: Read before write. Search memory BEFORE investigating to avoid re-discovering what's already known. Capture context and why, not just what. Make findings searchable with clear terminology. Verify accuracy before storing.

Use this skill after solving difficult bugs, discovering new patterns, investigating complex issues, or learning something that took significant time. Store all discoveries immediately to prevent context loss.
